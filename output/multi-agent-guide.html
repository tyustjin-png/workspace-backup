🤖 OpenClaw 多Agent架构方案

基于 OpenClaw 原生多Agent能力 · 一个Gateway · 多个专精Agent · 统一记忆

🎯 解决什么问题？

一个AI助手什么都管，看起来方便，但随着使用深入会遇到两个瓶颈：

问题表现
上下文膨胀对话越来越长，响应变慢，重要信息被稀释
人设漂移聊完投资接着聊健康，角色定位模糊，专业度下降
多Agent方案：每个Bot专精一个领域，像雇了一个专业团队。不同的事找不同的人聊。

🏗️ OpenClaw 原生多Agent能力

OpenClaw 原生支持

OpenClaw 从架构层面原生支持多Agent：一个Gateway进程可以同时运行多个独立Agent，每个Agent绑定不同的通信渠道（如不同的Telegram Bot）。这不是hack或变通方案，而是官方设计的核心功能。

原生提供的能力：

能力说明
agents.list 配置在 openclaw.json 中声明多个Agent，每个有独立ID和workspace
bindings 路由将不同渠道/账号的消息自动路由到对应Agent
独立 workspace每个Agent拥有独立的文件系统（SOUL.md、AGENTS.md、memory/）
独立 session每个Agent的对话历史完全隔离
共享 Gateway所有Agent共用一个进程、一套LLM配置、一个端口
独立 cron每个Agent可以有自己的定时任务
多渠道支持Telegram / Discord / WhatsApp / Signal / Slack 等均支持

架构示意：

Mac Mini（一台机器）
  └── OpenClaw Gateway（一个进程，端口 18789）
       ├── 🐉 紫龙（总管）@zilong_bot → workspace/
       ├── 🐕 二狗（投资）@jin_invest_bot → workspace-invest/
       └── 🧬 小尹烨（健康）@jin_health_bot → workspace-health/

用户体验：跟微信加了几个好友一样，不同的事找不同的人聊。

⚙️ 搭建步骤

1
创建Agent工作区 OpenClaw CLI
openclaw agents add invest
OpenClaw 自动创建独立的 workspace、session store、agent 目录。
2
创建Telegram Bot
在 Telegram 找 @BotFather，发送 /newbot，设置名字和用户名，拿到 Bot Token。
3
配置 openclaw.json OpenClaw 原生配置 
{
  "agents": {
    "list": [
      { "id": "main" },
      { "id": "invest", "workspace": "~/.openclaw/workspace-invest" }
    ]
  },
  "bindings": [
    { "agentId": "main",
      "match": { "channel": "telegram", "accountId": "default" } },
    { "agentId": "invest",
      "match": { "channel": "telegram", "accountId": "invest" } }
  ],
  "channels": {
    "telegram": {
      "accounts": {
        "default": { "botToken": "主Bot的token" },
        "invest":  { "botToken": "投资Bot的token" }
      }
    }
  }
}
4
重启Gateway + 配对 OpenClaw CLI 
openclaw gateway restart
openclaw pairing approve telegram <配对码>
给新Bot发消息会收到配对码，批准后即可使用。

🧠 记忆共享方案

核心挑战：多Agent各自workspace隔离，投资Bot不知道你跟健康Bot聊了什么。

第1层：共享核心记忆文件（推荐）

每个Agent的workspace放软链接，指向主Agent的 MEMORY.md：

ln -s ~/.openclaw/workspace/MEMORY.md \
     ~/.openclaw/workspace-invest/SHARED_MEMORY.md

所有Agent启动时都能读到你的核心画像、偏好和重要决策。

第2层：共享用户信息

将 USER.md 复制到每个Agent的workspace，让它们都认识你。

第3层（进阶）：Mem0 统一记忆服务

所有Agent共享一个记忆API，自动从对话中提取事实，支持三级作用域：

作用域说明
user级跨所有Agent共享，如个人偏好、基本信息
agent级单Agent专属，如投资Bot的持仓分析历史
session级单次会话，如当前讨论的上下文

大多数个人场景，前两层已足够。

📐 每个Agent的标准文件结构

OpenClaw 约定 每个Agent的workspace遵循统一规范： 
workspace-invest/
├── SOUL.md          # 人设、专业知识、行为规则
├── AGENTS.md        # 启动流程、工作规范
├── USER.md          # 用户信息（复制自主Agent）
├── SHARED_MEMORY.md # → 软链接到主Agent的MEMORY.md
├── TOOLS.md         # 工具使用备忘
└── memory/          # 独立记忆目录
    ├── YYYY-MM-DD.md
    └── ...

Agent启动时自动读取这些文件，建立身份认知和用户理解。

🤔 什么时候该拆Agent？

✅ 该拆❌ 不该拆
某领域对话量大（每天几十轮讨论）总对话量不大，一个Agent能hold住
需要完全不同的人设和专业知识库功能间关联性强（心情影响投资决策）
想让某Agent 24小时自动跑任务不想维护多套人设文件
核心原则："一个聪明的全才 + 几个专精助手" 优于 "多个割裂的专才"
