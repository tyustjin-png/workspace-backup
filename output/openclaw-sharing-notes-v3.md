## 一、省钱提效——模型分级 + 脚本优先

两个原则叠加，效果翻倍：**不是所有任务都需要最强模型**，**不是所有任务都需要AI**。

### 模型当员工分级

| 层级 | 模型 | 价格 | 适用场景 |
|------|------|------|----------|
| 执行层 | Haiku | $0.25/M | 固定逻辑、跑脚本、格式化输出 |
| 处理层 | Sonnet | $3/M | 信息处理、日报生成、搜索总结 |
| 思考层 | Opus | $15/M | 深度分析、策略制定、创作任务 |

我们系统跑着18个定时任务。仔细一看，6个是纯固定逻辑——日程文件创建、定投执行、日记提醒。逻辑完全一样，不需要任何"智能"。从Sonnet切Haiku，**日常成本降60%以上**，效果零差异。

### 任务三层金字塔

问自己三个问题：
1. **输入输出确定吗？** → 脚本
2. **需要LLM理解，但一步完成？** → Skill
3. **需要多步推理、动态调整？** → Agent

实际对照：
- 定投执行：读指数→算金额→调API。逻辑完全确定 → **脚本**
- Web3雷达：脚本抓数据，只在"打分"环节用LLM → **混合**
- 1244篇笔记深度分析：逐篇阅读、提炼、发现跨篇章联系 → **必须Agent**

最常见的错误：把简单任务包装成Agent。定投如果让Agent来做，它可能"思考"一下决定今天不买了——这不是你想要的。**脚本的优势是确定性**，同样输入永远同样输出。

省下的钱给真正需要思考的任务上Opus，整体效果反而更好。

## 二、知识蒸馏——从1244篇笔记到46条决策公理

### 背景

我从2017年开始用docall写笔记。7年下来积累了1244篇，涵盖投资决策、风险管理、市场分析、个人成长、内容创作——基本是我整个思维体系的原始记录。

问题是：**1244篇笔记，我自己都没全部回顾过。** 里面有多少被遗忘的洞察？多少重复踩的坑？多少其实已经形成但没被系统化的决策模式？

### 第一步：并行扫描——13个AI同时读

一个AI读1244篇要好几天。所以我用了并行分发：

**第一轮：Sonnet快速扫描**
- 1244篇分3批，3个Sonnet并行
- 几小时产出初步分析，发现问题先修

**第二轮：Opus深度阅读**
- 1244篇分13批，每批约100篇
- 13个独立Opus session并行跑
- 每批产出约20000字深度分析
- **一条指令 → 拆分13个AI并行 → 结果自动回报**

并行分发可能是个人用AI最被低估的能力。

### 第二步：三层蒸馏——从流水账到决策公理

```
L3 决策公理 (Axioms)     ← 最高抽象，指导决策
L2 模式提炼 (Reflector)  ← 从重复中识别规律
L1 日常观察 (Observer)   ← 每天事件流水账
```

从13份分析报告中提炼重复出现的模式。比如AI发现我在多篇笔记里反复强调"范围比精准更重要"——用网捕鱼而非叉刺鱼。这不是一条笔记，是一个横跨7年的决策模式。

最终从1244篇笔记中蒸馏出**46条决策公理**，7个类别：

| 类别 | 数量 | 示例 |
|------|------|------|
| 投资决策 | 9条 | "分区间定投优于无脑定投" |
| 风险管理 | 7条 | "永不满仓、不加杠杆、不做合约" |
| 信息处理 | 7条 | "一手信息优先，二手信息打折" |
| 人际关系 | 5条 | "看人看行动，不看嘴" |
| 个人成长 | 7条 | "行动在认知前面" |
| 内容创作 | 5条 | "能传播的文章，尽早发" |
| 团队管理 | 6条 | "管理不是约束，是减少摩擦" |

每条公理都标注了来源笔记编号、置信度和适用场景。经得起溯源检验。

### 关键洞察

- **信息密度提升几百倍**：1244篇 → 46条。找到信号，丢掉噪音。
- **大多数人卡在L1**：记了一堆笔记从来不回头看。AI可以帮你蒸馏——这可能是AI对个人知识管理最大的价值。
- **AI提炼必须人工审核**：它会过度泛化，也会编造不存在的规律。

## 三、AI越懂你，越能干活

前两条解决"怎么用AI"。这条解决"怎么让AI用得越来越好"。答案很简单：**把你的东西都给它，让它真正了解你。**

### 我做了什么

1. **1244篇笔记全部喂进去** — AI读完后，它知道我的投资逻辑、决策习惯、踩过的坑、性格特点
2. **46条公理写入系统** — AI做判断时不是从零推理，而是基于我的决策风格
3. **日常交互持续积累** — 每天对话、每次纠正、每个偏好，都在让AI更懂我

### 效果

- **给建议时带我的味道**。不是通用的"建议您分散投资"，而是"按你的INV-01公理，现在定投指数在3，该加仓了"
- **知道什么不该说**。它学会了我讨厌废话，讨厌"另一方面"，要的是结论+依据
- **能预判需求**。因为它知道我的日程习惯、工作节奏，能在我开口前准备好东西

### 本质

**传统AI助手：** 你问一句，它答一句。每次从零开始。

**喂过内容的AI助手：** 它带着对你的理解工作。你的思维模式、决策偏好、历史经验，全部在线。

这不是什么高深技术。就是**把你的内容交给AI，让它理解你，然后让它基于这个理解去干活**。你积累的笔记、文章、决策记录，本来就是你最有价值的数据。大多数人让它们躺在硬盘上发霉。现在可以激活它们。

## 四、每日日程系统——AI帮你管日程，不是提醒你管日程

日程不该是你每天手动打开的app，应该是AI主动推给你、帮你追踪、替你复盘的系统。

### 四个环节

**1. 自动创建 + 人工覆盖**
- 每天7:30脚本自动创建固定模板（运动/投资/写作/社交/家庭5大类）
- 如果我当天发了自己的日程，AI用我发的内容覆盖模板
- 固定模板是兜底，人的输入优先

**2. 语音/文字随时打卡**
- 对AI说"做了100个仰卧起坐"→ 自动归类到运动，标记✅
- 说"定投200U"→ 归类到投资
- 不需要打开任何app，对话里顺手就完成了

**3. 一天三次主动推送**
- 10:00 / 15:00 / 19:00 自动推送当前进度到Telegram
- 格式：✅已完成 / 🔸待完成 + 完成度统计
- 不是我去查日程，是日程来找我

**4. 晚上自动复盘**
- 20:00 每日总结：今天做了什么、完成率多少
- 21:00 审计反思：有没有漏掉的、明天需要调整什么

### 为什么有效

传统日程app：你得记得打开它 → 大多数人三天就忘了。AI日程系统：它主动追你 → 想忘都忘不了。本质是把"自律"外包给AI。你不需要意志力，系统帮你执行。

## 五、AI操作浏览器帮你卖二手——从查价到上架一条龙

AI不只是帮你写文案，它可以直接操控你的浏览器，帮你查价、写介绍、整理图片、完成上架。你动嘴，它动手。

### 背景

家里一台EPYC服务器要出掉，8个品类的配件：CPU、主板、64G/32G内存、多种规格的企业级SSD。

### AI实际干了什么

**1. 调用本地浏览器查市场价**
- AI打开浏览器，在闲鱼搜索同款配件
- 自动对比在售价格，给出挂价和底价建议
- 不是随便编个数——是真的去查了实时行情

**2. 编写上架介绍**
- 8个品类，每个都有完整文案
- 标题做了关键词SEO（"EPYC 7B12 64核128线程 无锁正式版"）
- 描述包含核心参数、卖点、使用时长、成色
- 还建议了套装打包方案（CPU+主板溢价10%）

**3. 整理图片**
- 列出每个配件需要拍的照片清单
- 正面全貌、背面金手指、标签特写、SMART截图
- 告诉你该拍什么、怎么拍

**4. 操作上架**
- AI直接在浏览器里操作闲鱼，填写商品信息
- 标题、描述、价格、分类，一步步完成
- 你在旁边喝茶就行

### 为什么这个例子有意思

大多数人用AI还停留在"帮我写个东西"。但AI能直接操控浏览器，意味着它可以**替你操作任何网页端的任务**——不只是生成内容，而是真的帮你干活。

8个品类如果自己查价+写文案+上架 → 半天起步。AI全程操作 → 你只需要拍照和确认。
