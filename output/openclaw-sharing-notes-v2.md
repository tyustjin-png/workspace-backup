## 一、省钱提效——模型分级 + 脚本优先

### 核心观点

两个原则叠加，效果翻倍：**不是所有任务都需要最强模型**，**不是所有任务都需要AI**。

### 模型当员工分级

| 层级 | 模型 | 价格 | 适用场景 |
|------|------|------|----------|
| 执行层 | Haiku | $0.25/M | 固定逻辑、跑脚本、格式化输出 |
| 处理层 | Sonnet | $3/M | 信息处理、日报生成、搜索总结 |
| 思考层 | Opus | $15/M | 深度分析、策略制定、创作任务 |

我们系统跑着18个定时任务。仔细一看，6个是纯固定逻辑——日程文件创建、定投执行、日记提醒。逻辑完全一样，不需要任何"智能"。从Sonnet切Haiku，**日常成本降60%以上**，效果零差异。

### 任务三层金字塔

```
        ┌─────────┐
        │  Agent  │  多步推理、动态策略
       ┌┴─────────┴┐
       │   Skill   │  LLM单步处理
      ┌┴───────────┴┐
      │    脚本     │  确定逻辑、固定流程
      └─────────────┘
```

问自己三个问题：
1. **输入输出确定吗？** → 脚本
2. **需要LLM理解，但一步完成？** → Skill
3. **需要多步推理、动态调整？** → Agent

**实际对照：**
- 定投执行：读指数→算金额→调API。逻辑完全确定 → 脚本
- Web3雷达：脚本抓数据，只在"打分"环节用LLM → 混合
- 1244篇笔记深度分析：逐篇阅读、提炼、发现跨篇章联系 → 必须Agent

最常见的错误：把简单任务包装成Agent。定投如果让Agent来做，它可能"思考"一下决定今天不买了——这不是你想要的。**脚本的优势是确定性**，同样输入永远同样输出。

**省下的钱给真正需要思考的任务上Opus，整体效果反而更好。**

## 二、知识蒸馏——从1244篇笔记到46条决策公理

### 背景：docall系统

我从2017年开始用docall（基于iA Writer）写笔记。7年下来积累了1244篇，涵盖投资决策、风险管理、市场分析、个人成长、内容创作——基本是我整个思维体系的原始记录。

问题是：**1244篇笔记，我自己都没全部回顾过。** 里面有多少被遗忘的洞察？多少重复踩的坑？多少其实已经形成但没被系统化的决策模式？

### 第一步：并行扫描——13个AI同时读

一个AI读1244篇要好几天。所以我用了并行分发：

**第一轮：Sonnet快速扫描**
- 1244篇分3批，3个Sonnet并行
- 几小时产出初步分析，发现问题先修

**第二轮：Opus深度阅读**
- 1244篇分13批，每批约100篇
- 13个独立Opus session并行跑
- 每批产出约20000字深度分析
- **一条指令 → 拆分13个AI并行 → 结果自动回报**

这一步的核心是**效率杠杆**。并行分发可能是个人用AI最被低估的能力。

### 第二步：三层蒸馏——从流水账到决策公理

```
L3 决策公理 (Axioms)     ← 最高抽象，指导决策
L2 模式提炼 (Reflector)  ← 从重复中识别规律
L1 日常观察 (Observer)   ← 每天事件流水账
```

**L1 → L2：识别模式**
从13份分析报告中提炼重复出现的模式。比如AI发现我在多篇笔记里反复强调"范围比精准更重要"——用网捕鱼而非叉刺鱼。这不是一条笔记，是一个横跨7年的决策模式。

**L2 → L3：凝结公理**
最终从1244篇笔记中蒸馏出**46条决策公理**，7个类别：

| 类别 | 数量 | 示例 |
|------|------|------|
| 投资决策 | 9条 | "分区间定投优于无脑定投" |
| 风险管理 | 7条 | "永不满仓、不加杠杆、不做合约" |
| 信息处理 | 7条 | "一手信息优先，二手信息打折" |
| 人际关系 | 5条 | "看人看行动，不看嘴" |
| 个人成长 | 7条 | "行动在认知前面" |
| 内容创作 | 5条 | "能传播的文章，尽早发" |
| 团队管理 | 6条 | "管理不是约束，是减少摩擦" |

每条公理都标注了来源笔记编号、置信度和适用场景。**经得起溯源检验。**

### 关键洞察

- **信息密度提升几百倍**：1244篇 → 46条。找到信号，丢掉噪音。
- **大多数人卡在L1**：记了一堆笔记从来不回头看。AI可以帮你蒸馏——这可能是AI对个人知识管理最大的价值。
- **AI提炼必须人工审核**：它会过度泛化，也会编造不存在的规律。公理是AI提出、人工确认的。

## 三、AI越懂你，越能干活

### 核心观点

前两条解决"怎么用AI"。这条解决"怎么让AI用得越来越好"。答案很简单：**把你的东西都给它，让它真正了解你。**

### 我做了什么

1. **1244篇笔记全部喂进去** — AI读完后，它知道我的投资逻辑、决策习惯、踩过的坑、性格特点
2. **46条公理写入系统** — AI做判断时不是从零推理，而是基于我的决策风格
3. **日常交互持续积累** — 每天对话、每次纠正、每个偏好，都在让AI更懂我

### 效果

蒸馏完成后，AI助手的表现有质的变化：

- **给建议时带我的味道**。不是通用的"建议您分散投资"，而是"按你的INV-01公理，现在定投指数在3，该加仓了"
- **知道什么不该说**。它学会了我讨厌废话，讨厌"另一方面"，要的是结论+依据
- **能预判需求**。因为它知道我的日程习惯、工作节奏，能在我开口前准备好东西

### 本质

**传统AI助手：** 你问一句，它答一句。每次从零开始。

**喂过内容的AI助手：** 它带着对你的理解工作。你的思维模式、决策偏好、历史经验，全部在线。

这不是什么高深技术。就是**把你的内容交给AI，让它理解你，然后让它基于这个理解去干活**。

你积累的笔记、文章、决策记录，本来就是你最有价值的数据。大多数人让它们躺在硬盘上发霉。现在可以激活它们。

### 踩坑经验

- **隐私边界要想清楚**。你愿意给AI看多少，提前想好。我是全给了，因为它是私有部署。
- **不要期望一步到位**。AI了解你是个渐进过程。第一天它只知道你写了什么，三个月后它知道你怎么想。
- **持续反馈很重要**。AI说错了就纠正，它的理解会越来越准。这本质上是在训练一个专属于你的判断模型。
