## 11:18 - API密钥管理机制设计（基于1Password）

### 📚 **学习来源**

**文章：** [还在把API Key发给你的龙虾么？别傻了](https://blog.yuanming.ai/posts/openclaw-1password-secrets/)

**核心发现：**
在Telegram中直接发送API Key会在**4个地方**留下明文痕迹：
1. Telegram云端服务器（永久存储）
2. LLM上下文窗口（经过模型提供商服务器）
3. 本地日志（~/.openclaw/logs/）
4. 配置文件（openclaw.json明文存储）

---

### ✅ **解决方案：OpenClaw SecretRef + 1Password**

#### 核心机制

```
1Password Agent Vault（保险柜）
    ↓ (Service Account Token)
同步脚本（搬运工）
    ↓
~/.openclaw/secrets.json（本地缓存，600权限）
    ↓ (File Provider)
OpenClaw SecretRef（钥匙卡）
    ↓
应用读取密钥
```

#### 迁移前后对比

**迁移前（openclaw.json）：**
```json
{
  "models": {
    "providers": {
      "anthropic": {
        "apiKey": "sk-ant-api03-xxx..."  ← 明文！
      }
    }
  }
}
```

**迁移后（openclaw.json）：**
```json
{
  "models": {
    "providers": {
      "anthropic": {
        "apiKey": {
          "source": "file",
          "provider": "op_file",
          "id": "/anthropic"  ← 引用！
        }
      }
    }
  }
}
```

---

### 📋 **实施方案（5步，30分钟）**

**Step 1: 安装1Password CLI**
```bash
brew install --cask 1password-cli
```

**Step 2: 1Password配置**
- 创建Agent Vault（专门存储AI密钥）
- 创建Service Account（无人值守，只读权限）
- 保存Token到 `~/.openclaw/.op-token`

**Step 3: 在Agent Vault中创建密钥**
- 在1P中（手机App最方便）创建Item
- 命名规范：`anthropic`、`brave`、`telegram-bot`、`gateway`

**Step 4: 同步脚本**
- 创建 `~/.openclaw/sync-secrets.sh`
- 自动从1P同步到本地JSON
- 注册File Provider

**Step 5: 迁移配置**
- 从非关键到关键逐个迁移
- 每次迁移后验证功能
- 备份 → 迁移 → 验证

---

### 🎯 **核心优势**

| 优势 | 说明 |
|------|------|
| **零明文存储** | openclaw.json只存引用，不存密钥 |
| **集中管理** | 所有密钥在1P中统一管理 |
| **随时随地** | 手机App就能新增/修改密钥 |
| **安全隔离** | Agent Vault独立，只读权限 |
| **自动同步** | 运行脚本即可同步最新密钥 |
