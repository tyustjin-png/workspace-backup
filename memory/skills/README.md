## 🎯 核心理念

**从"我学到什么"到"我会什么"**

## 🔄 技能生命周期

### 1️⃣ 提取（Extraction）
**触发时机：** 每日反思（21:00）或实时识别

**提取规则：**
- 我解决了一个问题，且这个解决方案可能再次使用
- 我学会了一个新工具/命令/脚本
- 我优化了一个流程

**自动提取问题：**
- "今天我学会了什么可复用的技能？"
- "有哪些操作步骤可以标准化？"
- "哪些问题以后可能再遇到？"

### 2️⃣ 应用（Application）
**触发时机：** 遇到类似场景时

**检索机制：**
- 语义匹配：当前情境 vs 触发条件
- 分类过滤：按 category 快速定位
- 效用排序：优先推荐高效用技能

**应用记录：**
```json
{
  "applied_at": "2026-03-09 10:30",
  "scenario": "金哥要求添加任务",
  "success": true,
  "feedback": "操作成功，用时2秒"
}
```

### 3️⃣ 进化（Evolution）
**更新触发：**
- 应用成功 → utility_score +0.1, usage_count +1
- 应用失败 → 记录失败原因，修订 execution_steps
- 连续10次成功 → 提升为"核心技能"
- 90天未使用 → 降级为"冷门技能"

## 📊 技能分类

| 类别 | 说明 | 示例 |
|------|------|------|
| **工具使用** | CLI/脚本/API的标准操作 | daily_schedule_manager.py |
| **问题诊断** | 排查问题的标准流程 | 检查cron日志、验证文件权限 |
| **数据处理** | 常见的数据转换/分析 | JSON解析、日志分析 |
| **通信协作** | 与金哥沟通的最佳实践 | 简洁汇报格式、风险提示 |
| **系统维护** | 服务器/环境管理 | 清理日志、重启服务 |

## 🎯 效用评分标准

| 分数 | 等级 | 说明 |
|------|------|------|
| 9-10 | 核心技能 | 每周使用多次，解决关键问题 |
| 7-8  | 常用技能 | 每周使用1-2次，效率提升明显 |
| 5-6  | 偶尔技能 | 每月使用几次，特定场景有用 |
| 3-4  | 冷门技能 | 很少使用，但保留备用 |
| 0-2  | 淘汰候选 | 几乎不用，考虑归档 |
