# 记忆系统架构（单一真相源）

## 🎯 设计原则

**统一入口，单一真相源** - 避免多套记忆系统互不通信

**核心理念：**
1. 所有记忆存储在 `memory/` 目录
2. 不使用多套系统（不同时用 MCP Memory Server 和文件系统）
3. 所有文件在 workspace 根目录可直接引用
4. AGENTS.md 作为启动指引，明确指向记忆位置

## 🧠 三层记忆模型

### 1️⃣ 记录层（Episodic Memory）
**文件：** `memory/YYYY-MM-DD.md`
**内容：** 原始经验记录
**特点：**
- 时间线组织
- 详细记录当天事件、对话、决策
- 保留上下文细节
- 原始材料，供后续提炼

**保留期限：**
- 活跃期：最近 7 天（每日反思时回顾）
- 归档期：7-90 天（每周反思时快速扫描）
- 清理期：> 90 天（提取核心认知后可删除）

---

### 2️⃣ 知识层（Semantic Memory）
**文件：** `lessons.md` / `patterns.md` / `rules.md` / `knowledge-graph.md`
**内容：** 从记录层提炼的抽象知识
**特点：**
- 去时间化（适用多场景）
- 结构化（便于检索）
- 动态更新（置信度衰减 + 效用反馈）
- 多维关联（知识图谱）

**更新机制：**
- 实时触发：犯错立即记录 lessons，第3次识别 patterns
- 定时批处理：每日 21:00 更新置信度，每周日清理过期

---

### 3️⃣ 核心层（Identity Memory）
**文件：** `MEMORY.md`（待创建）
**内容：** 策展的长期记忆
**特点：**
- 高度提炼（只保留最核心认知）
- 跨时间稳定（不轻易变动）
- 指导行为（塑造身份）
- 人类可读（金哥也能看懂）

**更新频率：**
- 每周日反思时筛选本周核心认知
- 每月末回顾，整合重复主题
- 重大认知突破时立即更新

## 🚫 不使用的方案

**不使用 MCP Memory Server：**
- 个人场景文件系统足够（金哥 + opus 4.6 确认）
- 避免多套系统冲突
- 保持简单可控

**不使用 Obsidian 等外部笔记系统：**
- 记忆就在 workspace，便于 Agent 直接读写
- 减少同步开销
- 单一真相源

**不使用向量数据库（当前阶段）：**
- 文件数量 < 1000，grep/memory_search 足够快
- 保持轻量，避免过度工程化
- 未来可选升级（如果文件量爆炸）

## 📊 系统健康指标

**每周检查（周日 21:00）：**
1. **碎片化检查**：是否有新的记忆文件出现在 `memory/` 外？
2. **冗余检查**：是否有重复内容分散在多个文件？
3. **活跃度检查**：patterns/rules 的"最近验证"时间分布
4. **衰减清理**：归档 > 90 天未验证的 patterns
5. **效用评分**：删除效用 < 3 的 rules

**健康标准：**
- ✅ 单一真相源（无碎片化）
- ✅ 信息流向清晰（记录 → 知识 → 核心）
- ✅ 动态更新（实时 + 定时双机制）
- ✅ 自我清理（衰减 + 效用反馈）
