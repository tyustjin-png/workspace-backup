# 记忆衰减检查机制

## 🎯 目的
自动检查 patterns.md 和 rules.md 中的"最近验证"时间，识别需要更新或归档的条目。

## ⏰ 执行频率
- **每周日晚上 21:00**（配合每日反思时间）
- 集成到每周反思流程中

## 📋 检查逻辑

### Patterns 衰减规则
```
最近验证时间距今：
- < 30天：活跃（无需处理）
- 30-60天：提醒验证（下次遇到相关场景时主动验证）
- 60-90天：置信度降为"待确认"
- > 90天：归档到 patterns-archived.md
```

### Rules 效用评分调整
```
基于应用记录自动调整：
- 连续 3 次应用效果好 ✓ → 评分 +1
- 出现效果差 ✗ → 评分 -2，触发修订
- 60 天未应用 → 评分 -1（逐步降权）
- 评分 < 3 → 归档或重新审视
```

## 🔄 执行流程

### 每周检查步骤
1. 读取 patterns.md 所有条目
2. 解析"最近验证"字段
3. 计算距今天数
4. 根据衰减规则分类：
   - 活跃：无需处理
   - 待验证：下周重点关注
   - 待确认：降低置信度
   - 待归档：移到 patterns-archived.md

5. 读取 rules.md 所有条目
6. 解析"应用记录"和"效用评分"
7. 根据应用频率调整评分
8. 生成周报：
   ```
   📊 本周记忆健康报告
   
   Patterns:
   - 活跃：X 条
   - 待验证：X 条（列出名称）
   - 降级：X 条（列出名称）
   - 归档：X 条（列出名称）
   
   Rules:
   - 高效用（8-10分）：X 条
   - 中效用（5-7分）：X 条
   - 低效用（<5分）：X 条（建议修订或归档）
   ```
