# 实时学习触发规则

## 🎯 目的
明确什么情况下立即更新记忆文件（不等晚上 21:00 定时反思）。

**核心原则：** 反思离错误越近，内化效果越好（ERL 研究）

---

## 🔥 触发条件与动作映射

### 1. 犯错时 → lessons.md
**触发信号：**
- 金哥批评/指出问题
- 任务执行失败/需要重做
- 发现自己的逻辑错误

**立即动作：**
1. 停止当前任务
2. 打开 `memory/lessons.md`
3. 写入新教训：
   ```
   ### LXXX - 教训名称
   **时间：** YYYY-MM-DD HH:MM
   **场景：** 做什么时犯的错
   **错误：** 具体做错了什么
   **原因：** 为什么会犯这个错
   **改进：** 以后怎么避免
   **相关规则：** 是否违反了已有规则
   ```
4. 如果是违反已有规则 → 同时更新 rules.md 的应用记录（标记 ✗）
5. 继续执行正确方案

**示例：**
- 金哥："这个回复太啰嗦" → 立即写入 lessons.md → 检查是否违反 R009

---

### 2. 同类事件第3次 → patterns.md
**触发信号：**
- 同一情况出现第 3 次
- 开始能预测金哥的反应/偏好
- 发现稳定的因果关系

**立即动作：**
1. 回顾前两次记录（从 daily logs 或 lessons 找证据）
2. 提炼为模式：
   ```
   ### 模式名称
   **观察次数：** 3次
   **首次发现：** YYYY-MM-DD
   **最近验证：** YYYY-MM-DD
   **描述：** 什么情况下会发生什么
   **应对策略：** 我应该怎么做
   **置信度：** 中（初次识别）
   ```
3. 如果可以提炼规则 → 同时写入 rules.md
4. 更新 knowledge-graph.md（建立关联）

**为什么是第3次？**
- 1次 = 偶然
- 2次 = 可能
- 3次 = 模式（统计学常用阈值）

---

### 3. 规则被应用 → rules.md 应用记录
**触发信号：**
- 遇到适用场景，应用了某条规则
- 应用后立即能看到效果（好/差）

**立即动作：**
1. 找到对应规则（如 R009）
2. 在"应用记录"字段追加：
   ```
   - 2026-03-08 10:55 应用，效果好 ✓（金哥未再批评废话）
   ```
3. 如果效果差：
   ```
   - 2026-03-08 应用，效果差 ✗（具体问题）
   → 需修订：建议改进方案
   ```
4. 根据应用记录调整效用评分（连续3次 ✓ → +1，出现 ✗ → -2）

---

### 4. 发现新关联 → knowledge-graph.md
**触发信号：**
- 讨论中突然意识到两个知识的联系
- 类比推理时发现相似/对比
- 一个决策依赖另一个知识

**立即动作：**
1. 识别关联类型（功能/因果/时序/决策）
2. 写入多维关联：
   ```
   ### 知识A ←→ 知识B
   ├── 📦 功能关系：相似/对比/互补
   ├── ⚡ 因果关系：A导致B / B依赖A
   ├── ⏰ 时序关系：先A后B / A演进为B
   └── 🎯 决策关系：选择依据
   ```
3. 跨领域迁移时特别重要（如"电脑普及"类比"AI助手普及"）

---

## ⚠️ 注意事项

### 什么时候不要立即记录？
- 不确定是否重要 → 先记到 daily log，晚上反思时再决定
- 信息还在变化中 → 等稳定后再写
- 噪音信息（琐碎细节）→ 不记录

### 如何避免过度打断？
- 实时记录要快（2分钟内完成）
- 只记关键要点，详细分析留给晚上
- 如果当前任务紧急，先在 daily log 标记 `TODO: 立即更新 lessons.md`

## 📊 效果追踪

**指标：**
- 实时记录次数 vs 晚上补记次数（目标：80% 实时）
- 规则应用记录完整度（目标：每次应用都记录）
- 模式识别速度（从发现到提炼的平均时间）

**每周检查：**
- 周日反思时回顾本周实时记录情况
- 调整触发阈值（如果第3次太晚，改为第2次）
