## 案例1：Quine NFT

### 📊 基本信息

| 项目 | Quine NFT |
|------|-----------|
| **背景** | Yuga Labs（无聊猿团队）项目 |
| **买入方式** | 公募抢购（荷兰拍） |
| **买入价格** | 7.6 ETH/个 |
| **买入时ETH** | ~$4,000 |
| **当前地板价** | 2.5 ETH |
| **当前ETH** | ~$2,000 |
| **账面亏损** | ~67% |

### 🧠 决策逻辑（当时的想法）

> "基本面很好，是无聊猿的团队做的"

### ❌ 错误分析

#### 错误1：纯团队背书，无内在价值
- NFT没有实际用途
- 完全依赖"大团队"光环
- 没有分析项目本身的价值主张

#### 错误2：没有止损
- 买入前没有设定退出条件
- 导致亏损从小扩大到-67%

#### 错误3：沉没成本谬误
- "觉得还会涨回来"
- 被套后不愿止损，越陷越深
- 忽略了：要涨回成本需要+200%，概率极低

#### 错误4：双重风险暴露
- NFT贬值（-34%）+ ETH贬值（-50%）
- 综合亏损-67%
- 没有考虑计价货币本身的风险

### 💡 教训提炼

1. **🔴 无实际用途 = 纯投机，风险极高**
2. **🔴 团队光环 ≠ 项目成功保证**
3. **🔴 买入时必须设止损（比如-30%就走）**
4. **🔴 套牢时问自己：如果是现金，我会买吗？不会就该卖**
5. **🔴 ETH计价资产有双重风险**

### 🎯 行动指南

**下次遇到类似情况时：**
1. 先问：这个项目有什么实际用途？
2. 再问：抛开团队，项目本身值这个价吗？
3. 买入前定好止损线（比如-30%）
4. 如果已经套牢，问：如果是现金，我会买吗？
