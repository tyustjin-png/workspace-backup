## 🔄 日程管理

### 日程文件位置
`/root/.openclaw/workspace/memory/daily-schedule/YYYY-MM-DD.md`

### 日程更新规则

**金哥发来日程时：**
1. 读取现有日程文件（保留已完成标记）
2. **合并**金哥的新增内容
3. 不要覆盖已完成的任务

**任务完成时：**
1. 更新日程文件（标记 [x]）
2. 如果是业务任务 → 推送通知

### 日程管理器
`/root/.openclaw/workspace/daily_schedule_manager.py`

用法：
```bash
# 处理输入（自动分类+标记完成）
python3 daily_schedule_manager.py "定投相关完成"

# 查看摘要
python3 daily_schedule_manager.py --summary

# 查看提醒
python3 daily_schedule_manager.py --reminder
```

## ⚠️ 教训记录

### L020 - 覆盖而非合并导致数据丢失
**日期：** 2026-03-10  
**问题：** 金哥08:03发来日程，我重写文件覆盖了08:00定投的完成标记  
**根因：** 写文件用覆盖模式，没有先读取再合并  
**修复：** 收到日程时先读取现有文件，保留已完成标记，再合并新内容

### L021 - 子任务结果处理缺失
**日期：** 2026-03-10  
**问题：** 收到验证审计结果后选择NO_REPLY，没有推送给金哥  
**根因：** 把"业务完成"误判为"系统正常"  
**修复：** 业务任务直接推送，不经过main session判断

## 🔍 快速诊断

### 定投没收到通知？
1. 检查 `/root/.openclaw/workspace/xiaoding/data/dca_log.txt`
2. 检查 `openclaw cron runs e43ae640 --limit 5`
3. 检查日程文件是否标记完成

### 日程汇报没收到？

3. 检查delivery状态

### 信息任务没收到？

2. 确认delivery.to配置正确 (6645374783)
