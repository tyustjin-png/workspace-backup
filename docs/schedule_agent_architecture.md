## 核心组件

### 1. 主Agent（紫龙）
- **模型**：Claude Sonnet-4.5
- **职责**：
  - 接收所有用户输入
  - 智能判断消息类型
  - 调用日程助手或直接回复
  - 统一返回结果

### 2. 日程助手模块
- **实现**：`schedule_assistant_wrapper.sh`
- **职责**：
  - 接收主Agent转发的输入
  - 调用分类脚本
  - 返回处理结果
  - 记录操作日志

### 3. 分类引擎
- **实现**：`daily_schedule_manager.py`
- **职责**：
  - 关键词匹配（评分制）
  - 任务分类（5大类）
  - 完成状态判断
  - 文件读写

### 4. 数据存储
- **位置**：`memory/daily-schedule/YYYY-MM-DD.md`
- **格式**：Markdown 清单格式
- **兼容**：Apple 备忘录

## 判断逻辑

主Agent 判断是否调用日程助手：

```python
# 伪代码
def is_schedule_related(user_input):
    schedule_keywords = [
        # 完成词
        '做了', '跑了', '写了', '定投了', '完成了',
        # 分类词
        '运动', '健身', '跑步', '投资', '定投',
        '邮件', '公众号', '推特', '午餐', '聚会',
        '罗老师', '父母', '儿子'
    ]
    
    # 关键词匹配
    for keyword in schedule_keywords:
        if keyword in user_input:
            return True
    
    # 特殊指令
    if '日程' in user_input or '待办' in user_input:
        return True
    
    return False
```

## 工作流程

### 用户输入："做了50个俯卧撑"

1. **主Agent接收**
   ```
   输入：做了50个俯卧撑
   判断：包含"做了" → 日程相关 ✅
   ```

2. **调用日程助手**
   ```bash
   schedule_assistant_wrapper.sh "做了50个俯卧撑"
   ```

3. **日程助手处理**
   ```bash
   python3 daily_schedule_manager.py "做了50个俯卧撑"
   ```

4. **分类引擎分析**
   ```
   关键词：做了(完成词) + 俯卧撑(运动词) = 2分
   分类：运动 ✅
   状态：已完成 ✅
   ```

5. **记录到文件**
   ```markdown
   ## ⛹️ 运动
   - [x] 做了50个俯卧撑
   ```

6. **返回确认**
   ```
   ✅ 已记录到【运动】并标记完成
   ```

7. **主Agent转发**
   ```
   ✅ 已记录到【运动】并标记完成：做了50个俯卧撑
   ```
