# 新项目启动向导

## 触发词

当用户说 `新项目` 时，启动此向导。

## 交互流程

### 步骤 1: 项目名称
```
🚀 启动新项目向导

1️⃣ 项目名称是？
   （例如：智能客服系统、网站重构、数据分析平台）
```

等待用户回复 → 保存为 `{PROJECT_NAME}`

---

### 步骤 2: 项目描述
```
2️⃣ 一句话描述这个项目？
   （例如：基于 GPT-4 的客服机器人）
```

等待用户回复 → 保存为 `{DESCRIPTION}`

---

### 步骤 3: 主要目标（可选）
```
3️⃣ 主要目标是什么？
   
回复目标，或直接说"跳过"让我自动补充。
   （例如：自动处理 80% 客户咨询、提升 50% 转化率）
```

- 用户回复目标 → 保存为 `{GOALS}`
- 用户说"跳过" → 基于 `{DESCRIPTION}` 自动生成合理目标

---

### 步骤 4: 自动创建

```
✅ 项目创建中...

📁 已创建：
- projects/{PROJECT_ID}/
  ├── PROJECT.md
  ├── README.md
  ├── tasks/
  │   ├── backlog.md
  │   ├── in-progress.md
  │   └── done.md
  ├── progress.md
  └── decisions.md

🎯 下一步建议：
1. 添加第一个任务：`新任务：[任务名称]`
2. 查看项目：`查看项目`
3. 切换回其他项目：`切换项目：[项目名]`

项目已启动，开始工作吧！🐉
```

---

## 变量替换规则

| 变量 | 来源 | 示例 |
|------|------|------|
| `{PROJECT_NAME}` | 用户输入 | 智能客服系统 |
| `{PROJECT_ID}` | 自动生成 (kebab-case) | smart-customer-service |
| `{DESCRIPTION}` | 用户输入 | 基于 GPT-4 的客服机器人 |
| `{GOALS}` | 用户输入或 AI 生成 | 自动处理 80% 咨询 |
| `{CREATED_DATE}` | 当前日期 | 2026-02-08 |
| `{STATUS}` | 默认 | 🚧 规划中 |
| `{PROGRESS}` | 默认 | 0 |

---

## 模板应用

使用 `projects/.template/` 中的模板，替换所有 `{变量}` 后生成新项目文件。
