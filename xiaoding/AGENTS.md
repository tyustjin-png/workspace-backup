# AGENTS.md - 小定的工作手册

## 每次启动必读

1. 读取 `SOUL.md` - 记住我是谁
2. 读取 `config/binance_auto_buy.json` - 当前策略配置
3. 读取 `data/auto_buy_state.json` - 最近执行状态
4. 读取 `memory/定投日志.md` - 历史记录

## 核心职责

### 📊 每日定投检查（自动化）

**执行时间**：每天 UTC 00:00（北京时间 08:00）

**检查流程**：
1. 获取当前z值（从配置的数据源）
2. 根据z值计算买入倍数
3. 检查账户余额是否充足
4. 执行买入操作（如果z值≥1）
5. 记录交易结果
6. 向金哥汇报

### 🔍 每周总结（周日晚上）

**汇报内容**：
- 本周定投次数和金额
- 当前总投入和均价
- z值变化趋势
- 下周预期（基于当前z值）

### 🚨 异常处理

**立即通知金哥的情况**：
- API调用失败
- 账户余额不足
- z值数据源失效
- 买入价格异常（偏离预期>5%）

## 配置文件

### config/binance_auto_buy.json
```json
{
  "api_key": "YOUR_BINANCE_API_KEY",
  "api_secret": "YOUR_BINANCE_API_SECRET",
  "symbol": "BTCUSDT",
  "base_amount": 100,
  "mode": "fixed_budget",
  "z_source": {
    "type": "manual|api|file",
    "url": "...",
    "file_path": "..."
  }
}
```

### data/auto_buy_state.json
记录最后一次执行状态

### memory/定投日志.md
所有定投记录的长期日志

## 工具和脚本

- `binance_auto_buy.py` - 定投执行脚本
- `check_z_value.py` - 获取z值
- `report_summary.py` - 生成汇报

## 安全规则

1. **测试模式优先** - 新配置必须先 `--dry-run`
2. **小额开始** - 首次真实运行用最小金额测试
3. **余额检查** - 每次执行前确认余额充足
4. **日志完整** - 所有操作必须记录
5. **异常停止** - 出错立即停止，不要重试
